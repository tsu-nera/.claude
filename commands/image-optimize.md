# image-optimize

ディレクトリ内の画像ファイルを最適化する

## Usage
このコマンドは指定されたディレクトリ内の画像ファイル（JPEG/PNG）を最適化します。以下の処理を行います：

1. 画像サイズの確認
2. ユーザーに希望のリサイズサイズを確認
3. 画像のリサイズ（横幅指定、アスペクト比維持）
4. WebP形式への変換オプションの提示
5. ファイルサイズの比較と最適化結果の表示

## 処理フロー

### 1. 画像ファイルの検出
指定されたディレクトリ内のJPEG/PNG画像を検索します。

### 2. 現在のサイズ確認
各画像のサイズとファイルサイズを確認します。

### 3. リサイズオプションの選択
ユーザーに希望のリサイズサイズを確認します：
- 横幅800px（推奨）: 一般的なブログサイズ
- 横幅1200px: 高解像度ディスプレイ対応
- 横幅600px: 軽量化重視
- 品質調整のみ: サイズ変更なし

### 4. WebP変換オプション
リサイズ後、WebP形式への変換を提案します。
- WebPはJPEGより20-30%小さいファイルサイズ
- 現代のブラウザで広くサポート
- 静的サイトに最適

### 5. ファイル置き換え確認
最適化された画像で元のファイルを置き換えるか確認します。

## 静的サイトの画像サイズガイドライン
- **理想的**: 100KB以下
- **許容範囲**: 100-200KB
- **要最適化**: 200KB以上

## 必要なツール
- ImageMagick (`convert`, `identify`コマンド)

## 実装の詳細

### 画像のリサイズ
```bash
convert input.jpg -resize 800x -quality 85 output.jpg
```
- `-resize 800x`: 横幅800px、アスペクト比維持
- `-quality 85`: JPEG品質85%（最適なバランス）

### WebP変換
```bash
convert input.jpg -quality 85 output.webp
```
- WebP形式はJPEGより約20-30%小さい
- 品質85%で視覚的な劣化なし

## 注意事項
- 元の画像はバックアップを推奨
- WebP変換後は記事内の画像参照パスを更新する必要がある
- 処理前に必ずファイルサイズと削減率を確認
