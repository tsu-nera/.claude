# コーディング規約

プロジェクト横断で適用するコーディング規約。
実装時・レビュー時の両方で参照する。

## ファイルサイズ（テストコード除く）

- **目標: 300行以内** — AIが最も正確に扱える粒度
- **上限: 500行** — 超えたら分割必須
- 既存の超過ファイルは修正の機会に分割を検討する（専用のリファクタリングIssueは立てない）
- 分割がIssueのスコープを膨らませるなら無理にやらない

## 品質

- 設定値はハードコード禁止（設定ファイルに格納）
- deprecated関数は完全削除（コメントアウト禁止）
- fallback処理は実装せず、エラーを上げるかログを残す

## リファクタリング

- 不要なコードはコメントアウトやdeprecatedではなく完全に削除
- 不要な変数の `_` リネームや `// removed` コメントは不要
